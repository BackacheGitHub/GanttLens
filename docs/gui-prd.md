# GanttLens GUI 模块 — 产品需求文档 (PRD)

## Problem Statement

GanttLens 目前只有 CLI 界面，用户必须通过命令行操作。对于项目管理者和团队成员来说，缺少一个直观的可视化界面来：
- 以甘特图形式查看项目进度
- 交互式地管理任务（修改日期、负责人、依赖关系）
- 直观地识别关键路径和资源过载
- 在同一个界面中编辑 PlantUML 源码并实时查看结果

## Solution

在现有 JavaFX GUI 模块基础上，实现一个混合架构的桌面应用：
- **左侧**：PlantUML 源码编辑器
- **右侧**：Canvas 渲染的交互式甘特图 + 分析面板
- 支持临时编辑 + 显式保存的工作流
- 提供任务属性编辑、依赖管理、资源分析等功能

## User Stories

### 文件与项目管理
1. As a 项目经理, I want to 通过文件选择器打开 .puml 文件, so that 我可以加载已有的项目计划
2. As a 项目经理, I want to 在左侧编辑器中直接编写 PlantUML 源码, so that 我可以快速修改计划
3. As a 项目经理, I want to 点击"解析"按钮后右侧甘特图实时更新, so that 我可以立即看到修改效果
4. As a 项目经理, I want to 保存修改后的计划到 .puml 文件, so that 我可以持久化我的工作
5. As a 项目经理, I want to 同时管理多个项目文件, so that 我可以对比不同项目的排期

### 甘特图可视化
6. As a 项目经理, I want to 看到每个任务显示为水平条形图, so that 我可以直观了解任务时间跨度
7. As a 项目经理, I want to 任务条用不同颜色表示状态（未开始/进行中/已完成）, so that 我可以一眼识别项目整体进度
8. As a 项目经理, I want to 任务条上有进度填充（如 60% 完成度）, so that 我可以用进度条而非数字百分比来判断完成情况
9. As a 项目经理, I want to 看到任务之间的依赖箭头, so that 我可以理解任务间的先后关系
10. As a 项目经理, I want to 看到时间轴（日期刻度）, so that 我可以精确定位任务在日历上的位置
11. As a 项目经理, I want to 关键路径上的任务有高亮标识, so that 我可以快速识别哪些任务延期会影响项目整体
12. As a 项目经理, I want to 鼠标悬停任务条时显示任务详情（名称、日期、负责人、进度）, so that 我可以快速获取信息而不离开当前视图
13. As a 项目经理, I want to 点击任务条时选中该任务, so that 我可以在属性面板中查看和编辑它
14. As a 项目经理, I want to 甘特图支持水平滚动, so that 我可以查看跨度数月的长时间线
15. As a 项目经理, I want to 甘特图支持垂直滚动, so that 我可以浏览 200+ 个任务而不卡顿

### 任务编辑与管理
16. As a 项目经理, I want to 点击选中任务后在属性面板中编辑任务名称, so that 我可以修正任务描述
17. As a 项目经理, I want to 通过日期选择器修改任务开始/结束日期, so that 我可以精确调整排期
18. As a 项目经理, I want to 通过下拉框修改任务负责人, so that 我可以重新分配资源
19. As a 项目经理, I want to 通过下拉框添加/移除任务依赖, so that 我可以调整任务间的先后关系
20. As a 项目经理, I want to 通过滑块或输入框修改进度百分比, so that 我可以更新任务完成状态
21. As a 项目经理, I want to 撤销（Undo）我在界面上的操作, so that 我可以回退误操作
22. As a 项目经理, I want to 重做（Redo）已撤销的操作, so that 我可以恢复被撤销的更改
23. As a 项目经理, I want to 界面上的修改不立即写入文件, so that 我可以先预览效果再决定是否保存

### 分析与洞察
24. As a 项目经理, I want to 在统计面板中看到项目总人天数, so that 我可以评估项目规模
25. As a 项目经理, I want to 在统计面板中看到每个人的工作量分布, so that 我可以判断资源分配是否均衡
26. As a 项目经理, I want to 在统计面板中看到关键路径信息, so that 我知道该重点盯哪些任务
27. As a 项目经理, I want to 在统计面板中看到过载警告, so that 我可以及时发现谁被过度分配
28. As a 项目经理, I want to 看到资源平衡建议, so that 我可以快速找到调优方案

### 批量操作
29. As a 项目经理, I want to 批量选中多个任务, so that 我可以一次性修改它们的负责人或日期
30. As a 项目经理, I want to 通过菜单或快捷键复制/粘贴任务, so that 我可以快速创建相似的任务

## UI 布局与交互描述

### 整体布局（BorderPane + SplitPane）

窗口从上到下分为四层：

1. **菜单栏**（Top）：File | Edit | Analyze | View | Help
2. **三栏工作区**（Center，水平分割）：
   - **左侧 - 文件树**：显示已打开的 .puml 文件列表，支持折叠
   - **中左 - 源码编辑器**：PlantUML 代码编辑区，底部带 [Parse] 和 [Save] 按钮
   - **右侧 - Tab 面板**：包含三个标签页
     - Gantt Chart：Canvas 甘特图（主视图）
     - Statistics：项目统计概览
     - Workload：工作负载热力图（后续迭代）
3. **状态栏**（Bottom）：显示任务数、总人天、过载数、关键路径天数、未保存标记

### 甘特图 Tab 详细布局

甘特图 Tab 内部从上到下分为三个区域：

1. **工具栏**：[Zoom+] [Zoom-] [Fit to Window] [Filter] [Search] 按钮
2. **主内容区**（水平分割）：
   - **左列 - 任务名称列表**：显示所有任务名，与右侧 Canvas 同步垂直滚动
   - **右侧 - Canvas 绘图区**：
     - 顶部：时间轴（日期刻度，工作日/周末底色区分）
     - 主体：任务条形图 + 依赖箭头
     - 支持水平和垂直滚动
3. **属性面板**（底部，选中任务时展开）：
   - 名称：文本输入框
   - 开始/结束日期：日期选择器
   - 负责人：下拉选择框
   - 进度：滑块或百分比输入框
   - 依赖：标签列表 + [+ Add Dependency] 按钮

### 甘特图 Canvas 绘图区视觉规范

参考 OmniPlan 风格，功能性优先：

| 元素 | 视觉表现 |
|------|----------|
| 任务条（未开始） | 浅灰色矩形，无边框渐变 |
| 任务条（进行中） | 蓝色矩形 + 进度填充（深蓝/浅蓝对比） |
| 任务条（已完成） | 绿色矩形 |
| 关键路径任务 | 红色边框（2px） |
| 选中状态 | 高亮边框 + 轻微放大 |
| 依赖箭头 | 灰色细线 + 箭头，从前置任务右端到后续任务左端 |
| 时间轴 | 顶部日期刻度线，工作日/周末底色区分 |
| 里程碑 | 菱形标记 |
| 今天线 | 红色竖虚线 |

### 统计 Tab 布局

统计面板从上到下分为四个区块：

1. **项目概览卡片**：三列并排显示
   - 任务数（如 25）
   - 总人天（如 120.5）
   - 工期（如 45 天）
2. **关键路径**：显示关键路径任务链及总天数
3. **资源分布**：水平条形图展示每个人的工作量（人天），按降序排列
4. **过载警告**：列出过载人员、日期、负荷百分比，附带可操作的平衡建议

### 核心交互流程

#### 编辑任务属性
1. 用户点击甘特图上的任务条 → 任务高亮选中
2. 底部属性面板展开，显示该任务的当前属性
3. 用户修改属性面板中的字段（日期选择器、下拉框、滑块等）
4. 甘特图实时重绘反映修改（内存模型更新，不写入文件）
5. 状态栏显示 "未保存更改 *"
6. 用户点击 Ctrl+S 或菜单 "保存" → 变更序列化回 .puml 文件

#### 撤销/重做
1. 用户按 Ctrl+Z → 回退到上一步的内存模型状态 → 甘特图重绘
2. 用户按 Ctrl+Y → 恢复已撤销的更改 → 甘特图重绘
3. 撤销栈仅包含当前会话的编辑，保存不清空撤销栈

#### 源码编辑 → 甘特图同步
1. 用户在左侧编辑器中修改 PlantUML 源码
2. 按 Ctrl+Enter 或点击"解析"按钮
3. 右侧甘特图重新解析并渲染
4. 如果有语法错误，编辑器中显示错误提示，甘特图保持上次有效状态

## Implementation Decisions

### 架构：混合 Scene Graph + Canvas

- **应用框架层**（菜单、侧栏、对话框、属性面板、Tab 面板）使用 JavaFX Scene Graph，利用其布局管理和交互能力
- **甘特图绘图区**使用 JavaFX Canvas，获得 200+ 任务规模下的渲染性能和精确像素控制
- Canvas 作为 Scene Graph 中的节点嵌入 Tab 面板

### 分层架构

GUI 模块分为四个核心组件：

- **GanttLayoutEngine**：纯 Java 类，输入任务列表 + 视图配置（缩放级别、日期范围），输出每个任务的屏幕坐标和尺寸。这是主要的可测试接缝，遵循 Humble Object 模式
- **GanttCanvasView**：Canvas 渲染层，遍历布局结果调用 GraphicsContext 绘制命令。作为"哑终端"，不包含业务逻辑
- **GanttInteractionHandler**：处理鼠标事件，将屏幕坐标通过 LayoutEngine 的命中检测函数转换为任务选择，触发业务操作
- **TaskSelectionModel**：管理选中状态，与属性面板绑定

### 数据流

```
PlantUML 源码
    ↓ [GanttFileParser]
GanttSchedule (内存模型)
    ↓ [AnalysisEngine]
ProjectStats
    ↓ [GanttLayoutEngine]
List<TaskLayout> (坐标/尺寸)
    ↓ [GanttCanvasView]
Canvas 像素输出
```

### 编辑模型

- 所有界面修改先作用于内存中的 `GanttSchedule` 副本
- 支持 Undo/Redo（基于操作栈的 Command 模式）
- 仅在用户显式触发保存时，才将变更序列化回 PlantUML 源码
- 保存时整体重新序列化，不做增量 patch

### 依赖关系

- GUI 模块依赖 core 模块的 `GanttFileParser`、`AnalysisEngine` 及所有 model 类
- 新增 `javafx-graphics` 依赖以支持 Canvas 渲染
- 测试新增 `testfx` + `monocle` 依赖（无头 UI 测试）

## Testing Decisions

### 测试策略

采用分层测试，逻辑测试为主力，截图对比为辅助：

1. **GanttLayoutEngine 单元测试（主力）**
   - 纯函数测试：给定任务列表 + 配置，断言输出的坐标/尺寸正确
   - 边界测试：0 任务、1 任务、200+ 任务
   - 缩放级别测试：不同缩放比例下坐标计算正确性
   - 命中检测测试：给定鼠标坐标，断言返回正确的任务

2. **交互逻辑测试**
   - TaskSelectionModel 测试：选中/取消选中/多选状态管理
   - Undo/Redo 栈测试：操作序列正确回退和恢复
   - 编辑操作测试：修改日期/负责人/依赖后内存模型正确更新

3. **视觉回归测试（辅助）**
   - 使用 TestFX + Monocle 无头模式截取甘特图快照
   - 与基准截图对比，差异 > 1% 时标记失败并生成 diff 图
   - 差异部分供人工审核确认
   - 仅在固定测试数据 + 固定窗口尺寸下运行

### 测试原则

- 测试外部行为和计算结果，不测试实现细节
- GanttLayoutEngine 的测试不依赖 JavaFX 运行时，可在纯 JUnit 环境下执行
- 截图对比测试标记为 `@Tag("visual")`，可在 CI 中选择性跳过

## Out of Scope

本 PRD 不包含以下内容（计划在后续迭代中实现）：

- **工作负载热力图**：颜色编码的人员/日期矩阵视图
- **拖拽交互**：拖拽任务条调整日期、拖拽箭头建立依赖
- **多项目管理**：同时打开多个 .puml 文件并跨项目分析
- **协作功能**：多用户实时编辑、版本历史对比
- **Web 版本**：当前仅限桌面 JavaFX 应用
- **打印/导出 PDF**：已有 Excel 导出，PDF 后续考虑
- **甘特图到源码的反向生成优化**：保存时使用完整重新序列化，不保留用户原始格式和注释

## Further Notes

- 当前 Task model 缺少 `color` 字段，需在 core 模块中扩展以支持 `is colored in Green` 语法在 GUI 中的视觉映射
- TaskStatus 目前只有 PENDING/IN_PROGRESS/COMPLETED 三种状态，可考虑增加 BLOCKED 状态以支持阻塞可视化
- 属性面板的依赖选择下拉框需要能显示所有可依赖的任务（排除自身和会产生循环依赖的任务）
- 甘特图的缩放级别需要支持"日/周/月"三种时间粒度，不同粒度下时间轴的刻度和任务条的宽度不同
